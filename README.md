## Project Overview

The **AI Prompt Monitoring System** is a SQL-based mini project designed to simulate how modern organizations monitor and analyze prompts submitted to AI systems.

With the rapid growth of generative AI platforms such as **ChatGPT, Gemini, and Copilot**, organizations must monitor prompts to detect potential **security risks, sensitive data leaks, and malicious inputs**.

This project demonstrates how SQL databases can be used to:

-   Track AI prompts submitted by users
-   Store AI responses
-   Detect risky prompts automatically
-   Monitor AI model usage
-   Generate analytics reports

The entire project is implemented in **one SQL file**, making it simple and efficient for learning database concepts.

## Database Structure

### 1\. Users Table

Stores system users who interact with AI models.

**Fields:**

-   user\_id
-   username
-   email
-   role
-   created\_at

* * *

### 2\. AI Models Table

Stores information about AI systems used in the platform.

**Fields:**

-   model\_id
-   model\_name
-   provider
-   version

* * *

### 3\. Prompts Table

Stores prompts submitted by users to AI systems.

**Fields:**

-   prompt\_id
-   user\_id
-   model\_id
-   prompt\_text
-   risk\_level
-   created\_at

* * *

### 4\. Responses Table

Stores responses generated by AI models.

**Fields:**

-   response\_id
-   prompt\_id
-   response\_text
-   response\_time
-   created\_at

* * *

### 5\. Sensitive Keywords Table

Stores keywords used for detecting risky prompts.

**Fields:**

-   keyword\_id
-   keyword
-   risk\_type

* * *

### 6\. Audit Logs Table

Tracks security-related activities and monitoring logs.

**Fields:**

-   log\_id
-   prompt\_id
-   action
-   log\_time

## Trigger Functionality

A trigger automatically analyzes prompts before insertion.

Example:  
If a prompt contains keywords such as:

-   password
-   credit card
-   api key
-   private key

The system automatically marks the prompt as **High Risk**.

* * *

## Stored Procedure

The project includes a stored procedure:

`get_high_risk_prompts()`

This procedure retrieves all prompts that have been classified as **High Risk**.

5.  Run the provided queries to explore the system.
